============================================================
ヤオヨロクリッカー：実績システム (Achievements) 定義・指示書
============================================================

この文書は「achievements.json」を作成・編集するための厳密なルールを定義したものです。
1000種類の実績を、矛盾なく、世界観を保ちつつ量産するための基準となります。

------------------------------------------------------------
1. JSON基本構造 (Schema)
------------------------------------------------------------
各実績は以下のキーを持つオブジェクトとして記述してください。

{
  "id": "識別ID",            // 重複不可。カテゴリ_連番を推奨 (例: obs_100)
  "rank": "ランク",          // bronze, silver, gold の3種類
  "category": "カテゴリ",    // 判定ロジックの分類 (後述)
  "title": "実績名",         // 世界観に沿った名前
  "flavor": "フレーバー",     // 世界観重視の解説。獲得条件は書かない
  "condition": "判定式",     // プログラムが評価する論理式 (後述)
  "display_req": "表示条件",  // プレイヤーに見せる「獲得条件」のテキスト
  "show_progress": true,    // 進捗（プログレスバー）を表示するか (true/false)
  "next_id": "次のID",       // 段階的実績の場合、次に目指す実績IDを指定
  "hint": "ヒント",          // 条件未達時に表示するヒント（隠し実績用）
  "ref_source": "元ネタ"     // 内部管理用。作品名や神話名を正確に記載
}

------------------------------------------------------------
2. ランクの定義 (Rank Definition)
------------------------------------------------------------
■ bronze (銅)
  - 難易度：低。普通に遊んでいれば数時間〜数日で取れるもの。
  - 内容：チュートリアル的要素、序盤の累計数、一般的な時間帯プレイなど。
■ silver (銀)
  - 難易度：中。狙って数日〜数週間かかるもの。
  - 内容：やや不運な結果、中規模な累計数、少し珍しいキャラの収集など。
■ gold (金)
  - 難易度：高。数ヶ月かかるもの、または「奇跡」が必要なもの。
  - 内容：膨大な累計数、全種コンプ、極めて特殊な数値の一致、極限状態プレイ。

------------------------------------------------------------
3. 判定式に使用可能な変数 (Logic Variables)
------------------------------------------------------------
condition キー内で使用できるシステム変数一覧です。

[累計系]
- total_clicks      : ゲーム開始からの総観測（クリック）回数
- total_coins       : ゲーム開始からの総獲得霊貨
- total_gacha       : ゲーム開始からの総ガチャ回数
- total_sublimation : ゲーム開始からの総昇華（レベルアップ）回数
- play_time_sec     : 総プレイ秒数

[現在値系]
- current_coins     : 現在の所持霊貨
- current_rare_coins: 現在の所持神結晶
- max_cps           : 過去最高の秒間クリック数 (Clicks Per Second)
- current_power     : 現在の総合神力
- current_attr_pwr  : 各属性の神力 (power_fire, power_water 等)

[収集系]
- collected_types   : 図鑑の登録種類数 (0〜460)
- rarity_5_count    : 所持している★5の種類数
- has_card_id_X     : 特定の神(ID:X)を所持しているか (true/false)

[外部・特殊系]
- battery_level     : 端末のバッテリー残量 (0〜100)
- current_hour      : 現在の時刻 (0〜23)
- day_of_week       : 曜日 (0:日曜〜6:土曜)
- unlucky_streak    : ★3以上が連続で出ていない回数
- idle_time         : 何も操作していない時間（秒）

------------------------------------------------------------
4. 執筆時の禁止事項・厳守事項
------------------------------------------------------------
1. 【元ネタの正確性】
   アニメやゲームをパロディにする場合、作品の枠を超えて要素を混ぜることを厳禁とする。
   (例: 「無限の指製」にドラゴンボールのネタを混ぜるなど)
2. 【フレーバーテキストのトーン】
   フレーバーテキストは常に「内気なギターリスト兼・観測者」が記したような、
   神秘的で少しオドオドした、あるいは詩的な表現を保つこと。
   システム用語（クリック、レベル、ガチャ）は極力フレーバー内では言い換える。
   (例: クリック→観測、ガチャ→招集、霊貨→縁、など)
3. 【プログレスバーの整合性】
   show_progress を true にする場合、必ず condition は「数値 >= 目標値」の
   形式でなければならない。

------------------------------------------------------------
5. カテゴリ分類 (Category)
------------------------------------------------------------
- observation : 観測（クリック）に関するもの
- money       : 霊貨・神結晶に関するもの
- collection  : 図鑑・カード収集に関するもの
- luck        : ガチャの結果・運に関するもの
- time        : プレイ時間・時間帯・曜日に関するもの
- terminal    : 端末状態（バッテリー等）に関するもの
- system      : 隠しコマンド、UI操作などに関するもの
- myth        : 特定の神々にまつわるエピソード再現
- parodee     : アニメ、ネットミーム等のパロディ
